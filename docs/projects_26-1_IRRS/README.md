# 정보검색및추천시스템 기말 프로젝트 구상안

## 문제 정의

현재 오스트리아도서관 HUFS 검색 시스템의 한계:
- 정확한 제목을 알아야만 검색 가능 (완전 일치 기반)
- 언어 장벽: "괴테" 검색 시 "Goethe" 결과 미출력
- 장르/내용 기반 검색 불가 ("희곡" 검색 시 해당 장르 도서 미출력)

---

## 시스템 설계

### 1단계 — 임베딩 사전 계산 (오프라인)
- 전체 도서(제목 + 저자)를 **XLM-RoBERTa**로 임베딩 계산
- 계산된 벡터를 **FAISS 인덱스**로 저장
- 신규 도서 추가 시 해당 도서 임베딩만 계산해서 인덱스에 append → 재계산 불필요

### 2단계 — 검색 (온라인, 실시간)
1. **키워드 매칭** (최우선): 제목/저자 정확 매칭
2. **벡터 유사도 검색**: 쿼리를 XLM-RoBERTa로 임베딩 → FAISS로 유사 도서 검색
   - 번역 없이 한국어↔독일어 cross-lingual 검색 자동 지원
3. 키워드 결과 우선, 중복 제거 후 벡터 결과 추가하여 최종 반환

---

## 평가

- **Metric**: Precision@K
- **방법**: 샘플 쿼리셋 직접 제작 → 상위 K개 결과 관련성 수동 라벨링
- 키워드 매칭만 쓴 기존 시스템과 비교하여 개선 폭 측정

---

## 기술 스택

| 역할 | 기술 |
|---|---|
| 임베딩 모델 | XLM-RoBERTa (HuggingFace) |
| 벡터 검색 | FAISS |
| 백엔드 연동 | Django REST API (기존 시스템) |
